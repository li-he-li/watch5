## 📋 项目概述

**项目目标：** 开发一个完整的三端心率监测系统，实现从手表采集数据到手机中转，再到电脑实时显示和分析的完整数据流。

**系统架构（针对Galaxy Watch 5优化）：**
```
┌─────────────────────┐    Data Layer API    ┌─────────────┐    WiFi/BLE     ┌─────────────┐
│  Galaxy Watch 5     │ ─────────────────→  │  Android    │ ─────────────→  │  Desktop    │
│  (Wear OS 4.0)      │   (推荐方式)        │  Phone      │   优先WebSocket │  App        │
│                     │                     │             │   备用BLE       │             │
│  ⭐ BioActive传感器 │                     │  🔄 数据中转 │                │  📊 可视化  │
│  ⭐ 动态采样率      │                     │  ⚡ 双传输  │                │  💾 分析   │
└─────────────────────┘                     └─────────────┘                └─────────────┘
```

**架构说明（基于设备特性）：**
1. **手表→手机**：Data Layer API（Wear OS官方推荐，最稳定）
2. **手机→电脑**：
   - **优先方案**：WebSocket（跨WiFi，距离无限制）
   - **备用方案**：手机作为BLE GATT Server广播（短距离10-20米）
3. **为什么不用手表直接BLE广播？**
   - ⚠️ Galaxy Watch 5的BLE Peripheral模式需要特殊证书
   - ⚠️ 后台BLE广播会大幅增加电池消耗
   - ✅ Data Layer API经过优化，功耗低且稳定

**关键决策：** 三个应用需要**分开开发**，但可以通过Kotlin Multiplatform实现**业务逻辑共享**。

class HeartRateSensorManager(private val context: Context) {
    private val sensorManager = context.getSystemService(Context.SENSOR_SERVICE) as SensorManager
    private val heartRateSensor = sensorManager.getDefaultSensor(Sensor.TYPE_HEART_RATE)

    // 动态采样率：静止1Hz，运动5Hz
    private var currentSamplingRate = SensorManager.SENSOR_DELAY_NORMAL

    fun startListening() {
        sensorManager.registerListener(
            sensorEventListener,
            heartRateSensor,
            currentSamplingRate
        )
    }

    // 根据运动状态动态调整采样率
    fun adjustSamplingRate(isMoving: Boolean) {
        currentSamplingRate = if (isMoving) {
            SensorManager.SENSOR_DELAY_UI // 60ms (约16Hz)
        } else {
            SensorManager.SENSOR_DELAY_NORMAL // 200ms (5Hz)
        }
    }
}

// Data Layer通信（Wear OS官方推荐）
class WearDataSyncManager(private val context: Context) {
    private val dataClient = Wearable.getDataClient(context)

    suspend fun syncHeartRateData(heartRate: Int) {
        val dataMap = DataMapBuilder()
            .putInt("heartRate", heartRate)
            .putLong("timestamp", System.currentTimeMillis())
            .putInt("battery", getBatteryLevel())
            .build()

        val request = PutDataRequest.create("/hr-data").apply {
            data = dataMap.toByteArray()
        }

        dataClient.putDataItem(request)
    }
}

[手机 - 数据处理]
    │
    ├─ 数据验证和过滤
    ├─ 时间戳对齐
    ├─ 异常值检测
    │
    ↓
[传输方式选择]
    │
    ├─ 优先：WebSocket
    │   └─→ 电脑（WebSocket Server）
    │
    └─ 备用：BLE GATT Server (0x180D)
        └─→ 电脑（BLE Central）
    │
    ↓
[电脑 - 接收与展示]
    │
    ├─ 实时图表更新
    ├─ 数据存储（SQLite）
    ├─ 统计分析
    └─ 异常告警

data class HeartRateData(
    val timestamp: Long,           // Unix时间戳（毫秒）

    val deviceId: String,          // 设备唯一标识

    val signalQuality: Int? = null // 信号质量（可选）

// BLE GATT格式（符合0x2A37标准）
data class BleHeartRateMeasurement(

## ⚠️ 关键技术挑战与解决方案

### 挑战1: Wear OS后台运行限制

**问题：** Wear OS对后台应用有严格限制

**解决方案：**
```kotlin
// 使用前台服务
class HeartRateMonitorService : Service() {
    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        // 创建通知
        val notification = NotificationCompat.Builder(this, CHANNEL_ID)
            .setContentTitle("心率监测中")
            .setSmallIcon(R.drawable.ic_heart)
            .setOngoing(true)
            .build()

        startForeground(1, notification)

        // 请求电池优化白名单
        requestIgnoreBatteryOptimizations()

        return START_STICKY
    }
}
```

---

### 挑战2: BLE广播距离限制

**问题：** BLE典型距离仅10-20米

**解决方案：**
1. **优先WebSocket：** 通过WiFi传输，无距离限制
2. **备用BLE：** 用于短距离场景（如电脑在同一房间）
3. **自动切换：**
```kotlin
class TransmissionManager {
    suspend fun sendData(data: HeartRateData) {
        when {
            bleClient.isConnected() -> bleClient.send(data)
            websocketClient.isConnected() -> websocketClient.send(data)
            else -> cache(data) // 离线缓存
        }
    }
}
```

**问题：** 华为等设备BLE兼容性差
